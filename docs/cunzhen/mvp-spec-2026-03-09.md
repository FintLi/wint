# 村圳 MVP 产品规格

更新日期：2026-03-09

## 1. 产品定位

村圳 v1 不是开放大市场，而是“精选真房源 + 快速匹配 + 带看成交”的租房助手。

### 目标用户

- 租客：在深圳宝安西乡 / 福永通勤带工作的青年租客
- 供给方：合作中介、二房东、房东、兼职拍房员
- 运营方：你自己或小团队

### 首期范围

- 预算：`800-2500` 元
- 房型：`单间 / 合租 / 一房`
- 区域：宝安 `3-5` 个村

## 2. 三个界面

## 租客端

只做 6 个能力：

- 房源列表
- 通勤 / 预算筛选
- 房源详情
- 求租卡
- 预约看房
- 客服接管

## 供给端

- 邀约式录入
- 不开放全民自助发布
- 只允许合作方提交简表
- 所有房源必须经过审核后上架

## 运营端

- 以成交漏斗为核心
- 管理房源状态、线索状态、带看状态、结算状态
- 管理失效房源和版本记录

## 3. 核心页面

## 3.1 房源列表

页面目标：让租客在 `30-60` 秒内找到愿意点开的房源。

展示字段：

- 标题
- 村名 / 片区
- 预算
- 房型
- 可入住时间
- 标签：近地铁、采光好、女生友好、夜归主路等
- 最近核验时间
- 认证状态

筛选条件：

- 预算区间
- 上班地方向 / 通勤片区
- 房型
- 是否电梯
- 入住时间

## 3.2 房源详情

详情页必须展示：

- 租金
- 押付方式
- 水电规则
- 楼层 / 采光
- 是否电梯
- 家具家电
- 可入住时间
- 最近核验时间
- 实拍视频
- 安全 / 避坑标签

详情页 CTA：

- 预约看房
- 提交求租卡
- 联系客服

## 3.3 求租卡

必填字段：

- 姓名
- 联系方式
- 预算下限
- 预算上限
- 上班地
- 最晚入住时间
- 房型偏好

可选字段：

- 性别
- 是否接受合租
- 电动车需求
- 对噪音 / 夜归 / 独卫的要求

## 3.4 预约看房

状态固定为：

- `待确认`
- `已确认`
- `已完成`
- `爽约`

每次爽约必须记录原因。

## 3.5 供给录入

供给方提交字段：

- 房源标题
- 村名 / 片区
- 房型
- 租金
- 押付方式
- 水电规则
- 楼层
- 采光
- 是否电梯
- 家具家电
- 可入住时间
- 实拍视频链接
- 联系人身份
- 联系方式

录入后进入 `待核验`，不能直接前台展示。

## 3.6 运营后台

线索状态固定为：

- `新线索`
- `已联系`
- `已匹配`
- `已带看`
- `已成交`
- `流失`

房源状态固定为：

- `待核验`
- `可发布`
- `已预约`
- `已成交`
- `已失效`

## 4. 关键规则

## 4.1 房源上架规则

任一字段缺失则不能上架：

- 租金
- 费用规则
- 最近核验时间
- 实拍视频

## 4.2 72 小时刷新规则

- 房源核验时间超过 `72` 小时自动转为 `已失效`
- 失效房源不在前台展示
- 刷新时记录最新版本，不覆盖历史

## 4.3 版本记录规则

以下字段变化必须生成新版本：

- 租金
- 可入住时间
- 联系方式
- 水电规则
- 认证状态

前台只展示最新有效版本，后台保留历史版本。

## 4.4 隐私与安全规则

- 前台不展示精确门牌
- 不展示可绕过平台结算的完整联系方式
- 只收集成交所需最少信息

## 5. 开发优先级

## P0：不开发也能跑通

- 内容获客
- 表单收集求租卡
- 表格管理房源和带看
- 微信 / 企业微信接管客服

## P1：最小小程序壳

- 房源列表
- 房源详情
- 求租卡
- 预约看房
- 客服接管

## P2：供给端邀约录入

- 简表提报
- 审核上架
- 状态同步

## P3：运营后台产品化

- 漏斗看板
- 版本管理
- 结算管理
- 失效自动提醒

## 6. 验收标准

- 租客可在 `3` 分钟内提交完整求租卡
- 当天能收到至少 `3` 套匹配房源
- 任一不完整房源不能上架
- 超过 `72` 小时未复核房源自动失效
- 爽约原因可回写并复盘
- 每笔收费可追溯到线索来源和供给方
