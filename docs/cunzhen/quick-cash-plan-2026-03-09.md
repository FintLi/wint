# 村圳快速赚钱执行包

更新日期：2026-03-09

## 1. 一句话策略

先不做“大平台”，只做“宝安西乡 / 福永城中村真房源成交助手”：

- 用短视频、图文和社群内容把高意向租客引进来
- 用表单 / 企业微信 / 微信小程序承接线索
- 用人工核验、人工匹配、人工带看拿下首批成交
- 只向供给侧收费，不向租客收费

这套打法的核心不是技术，而是：**真房源、快响应、低踩坑、固定时段带看**。

## 2. 你现在最该追求的结果

### 14 天目标

- 拿到 `30` 套已核验房源
- 拿到 `40` 个有效求租线索
- 完成 `8` 次带看
- 成交 `1-2` 单
- 验证哪类内容最容易带来高意向租客

### 30 天目标

- `60` 套有效房源
- `90` 个有效求租线索
- `18` 次带看
- `4` 单成交
- 跑通供给侧收费

### 60 天目标

- `80` 套有效房源
- `150` 个有效求租线索
- `30` 次带看
- `8` 单成交
- 工作时间首响 `<10` 分钟
- 失效房源占比 `<15%`

## 3. 先卖什么，而不是先做什么

### 3.1 第一阶段只卖 1 个产品

产品名：`带看成交包`

- 面向对象：合作中介、二房东、房东
- 收费时点：成交后收费
- 收费方式：固定服务费，不抽租客佣金
- 价格建议：
  - `单间 / 合租`：`300-500` 元 / 单
  - `一房`：`600-800` 元 / 单

### 3.2 为什么先向供给侧收费

- 租客对“找房信息”天然不愿意先付费
- 供给侧更在意空置时间，愿意为更快成交付费
- 供给侧只要感知到“你带来的人靠谱、到场率高、签约快”，就愿意复购

### 3.3 第 20 单后再开的 3 个产品

- `认证房源包`：按周或按月收费，提供认证标识、更新时间展示、优先排序
- `急租加速包`：`3-7` 天内高频推送和固定带看时段
- `搬家 / 宽带 / 保洁转介`：按单收服务费或拿渠道返佣

## 4. 不要一开始就做的事

- 不做全深圳
- 不做开放式房东自由发布
- 不做在线签约和在线支付
- 不做复杂地图找房
- 不做“长租公寓品牌化改造”
- 不做没有视频、没有核验时间的房源展示
- 不在 `3` 单成交前开发复杂后台

## 5. 最小团队配置

你不需要组公司班底，先用 `1 + 2` 模式：

- 你自己：内容、线索转化、合作谈判、价格决策
- 兼职拍房员 `1` 人：按套采房、拍视频、录字段
- 兼职带看员 `1` 人：集中在固定时段带看

如果预算很紧，可以一开始只有你自己 + 一个兼职拍房员，带看先自己做。

## 6. 第一月建议预算

建议把预算控制在 `3000-8000` 元。

### 固定开支

- 交通和打车：`500-1200`
- 拍房补贴：`1000-2500`
- 手机号 / 企业微信 / 通讯：`100-300`
- 简单剪辑或封面制作：`300-1000`
- 杂费与请客沟通：`500-1500`

### 不建议的花销

- 先花几万做定制小程序
- 先租办公室
- 先投广泛广告
- 先找全职销售

## 7. 日常工作分配

### 周一到周五

- 上午：回线索、筛选高意向用户、安排当日带看
- 中午：发 `1` 条图文或短内容
- 下午：拍房、核验、更新房源
- 晚上：发 `1` 条短视频，集中回私信

### 周六到周日

- 固定看房时段
- 集中成交推进
- 盘点哪些内容带来的线索质量最高

建议固定两个带看窗口：

- `周三 19:30-22:00`
- `周六 14:00-18:00`

固定窗口能显著降低协调成本。

## 8. 0-14 天执行节奏

## 第 1-2 天：收窄战场

- 只选 `3-5` 个村，不要扩散
- 只服务 `800-2500` 元预算、单间 / 一房租客
- 建立 4 张表：房源表、求租表、带看表、内容表
- 确定统一标准字段，不完整的房源一律不上架

建议首批观察组合：

- `西乡` 通勤盘
- `固戍 / 后瑞` 通勤盘
- `福永` 性价比盘

## 第 3-4 天：先做访谈，不先做开发

- 访谈 `20` 个租客
- 访谈 `10` 个供给方
- 目标不是聊天，而是确认下面 5 件事：
  - 哪个预算带最活跃
  - 哪个上班地最常见
  - 租客最怕踩什么坑
  - 房东 / 中介最怕你什么事做不到
  - 成交后谁来付钱最自然

访谈结束就把话术、标签和内容选题写进模板，不要停留在聊天记录里。

## 第 5-7 天：拿首批供给

- 目标：首批 `30` 套已核验房源
- 配比建议：
  - `50%` 合作中介 / 二房东
  - `30%` 兼职拍房员采集
  - `20%` 房东私信邀约
- 每套房源必须有：
  - 租金
  - 押付方式
  - 水电规则
  - 楼层 / 采光
  - 是否电梯
  - 家具家电
  - 可入住时间
  - 最近核验时间
  - 实拍视频

没有视频，不上；超过 `72` 小时不刷新，下架。

## 第 8-10 天：开始发内容，不求爆款

每天发 `2` 条内容即可，不要追求花哨。

只发 3 类：

- `真实房间实拍`
- `预算 / 通勤选村攻略`
- `城中村避坑与安全分级`

每条内容只要一个 CTA：

- `私信“预算+上班地”`
- `点击表单填求租卡`
- `加微信领今日可看房源`

不要一条内容里放多个动作。

## 第 11-14 天：只追高意向线索

把线索分成 3 类：

- `A 类`：`7` 天内必须入住
- `B 类`：有明确预算和上班地
- `C 类`：只是看内容、暂时无明确计划

前两周只重点追 `A + B`，因为你要的是第一笔钱，不是粉丝数。

## 9. 15-30 天执行节奏

- 把日更内容固定成模板，不再每次临时想选题
- 把高意向用户引导到统一求租卡，不再靠私信散聊
- 每周至少做 `2` 次集中带看
- 每周清一次“假房源 / 失效房源 / 长期不回的合作方”
- 开始对合作供给方收第一笔成交费

### 这个阶段才做的小程序

如果前两周已经拿到 `1-2` 单或接近成交，再做最小小程序壳：

- 房源列表
- 房源详情
- 求租卡
- 预约看房
- 客服接管

供给录入和运营后台先继续用表格 / 表单 / 飞书多维表，不要急着开发。

## 10. 31-60 天执行节奏

- 确定最容易成交的 `1` 个预算带和 `1` 个村群组合
- 把不赚钱的片区和房型砍掉
- 给合作供给方分层：高质量、普通、待淘汰
- 给线索来源分层：高意向内容、普通内容、纯泛流量
- 开始提供 `认证房源包` 和 `急租加速包`

如果到第 `45` 天还没有稳定成交，优先调整这 4 个位置，而不是继续开发：

- 内容是否吸引了错误人群
- 房源是否不够真或不够新
- 带看时段是否过于分散
- 供给侧收费是否过早或过高

## 11. 内容策略

你有内容能力，所以这部分是最重要杠杆。

### 只做两个渠道

从你最顺手的渠道里选 `2` 个，不要同时铺太多。

- 一个做公域内容
- 一个做私域承接

建议优先顺序：

- 短视频 / 图文公域 `1` 个
- 微信私域 `1` 个

### 最容易转化的内容框架

#### 框架 1：真实房间实拍

- 开头直接报预算和位置
- 中段展示楼道、采光、卫生间、水电说明
- 结尾只给一个动作：填求租卡或私信关键词

#### 框架 2：通勤选村攻略

- 按上班地反推村
- 明确通勤时长、预算区间、适合人群
- 结尾引导“告诉我预算+上班地，我给你配 3 套”

#### 框架 3：避坑和安全分级

- 列出真实坑点
- 讲清楚如何判断水电、隔断、消防、夜路、噪音
- 强调你会过滤掉哪些房源

## 12. 供给侧合作策略

### 优先合作对象顺序

1. 小中介门店里的高执行经纪人
2. 二房东 / 包租方
3. 自己管理出租屋的个人房东

### 你给供给侧的价值表达

不要讲“我要做平台”，要讲：

- 我带的是高意向租客
- 我会先筛预算、上班地、入住时间
- 我只推真实在租的房子
- 我集中带看，减少你重复接待
- 成交后才收费

## 13. 第一笔钱怎么更快到来

最现实路径不是“等房东主动来发房”，而是：

1. 先拿 `20-30` 套真实房源
2. 发 `10-20` 条强筛选内容
3. 收集 `20-30` 个高意向求租线索
4. 安排 `5-8` 次带看
5. 成交 `1-2` 单

第一笔钱出现之后，再去谈更多合作方最容易。

## 14. 单位经济模型

建议你按下面这组数做判断：

- 平均服务费：`520` 元 / 单
- 兼职拍房和路费摊销：`80-180` 元 / 单
- 兼职带看摊销：`60-120` 元 / 单
- 单笔毛利目标：`250-350` 元 / 单

只要你能把成交周期压到 `7-10` 天，这个模型就有继续优化的空间。

但要注意一个很现实的结论：

- 如果你按偏低服务费收费
- 同时又把拍房、带看、交通和固定支出放大
- 那么即使做到 `8` 单，也可能只是“验证了需求”，但还没有真正赚钱

所以前 `60` 天的经营纪律要非常硬：

- 平均服务费尽量往 `600+` 拉，而不是长期停在低价
- 单个有效房源采集成本压到 `20-30` 元
- 单次带看履约成本压到 `60-90` 元
- 固定成本尽量压在 `3000` 元上下
- 带看尽量批量化，减少单次跑动

你仓库里的 `tools/cunzhen_unit_economics.py` 就是用来盯这件事的。每次想扩区、加人、加开发前，先跑一遍测算。

如果一单下来你净剩不到 `150` 元，说明要立刻优化：

- 片区太散
- 线索太泛
- 房源失效率太高
- 带看太分散

## 15. 红线和风险

- 不展示精确门牌和可直接绕过你的详细联系方式
- 不发布未核验、视频不全、价格不清的房源
- 不提前承诺“百分百能租到”
- 不在合规未确认前收预付款、订金或信息费
- 不收集超出必要范围的个人信息

## 16. 你醒来后第一天该做什么

按顺序做，不要跳。

1. 建 4 张表，用仓库里的模板直接开干
2. 写一版供给侧合作话术
3. 写一版租客求租表单
4. 约 `5` 个供给方和 `10` 个租客访谈
5. 确定 `3-5` 个村
6. 拍或收第一批 `10` 套房的视频
7. 发第一批 `3` 条内容
8. 当天晚上一口气回完所有私信

## 17. 最后一句话

你这个项目能不能快速赚钱，决定因素不是“小程序做得多漂亮”，而是你能不能在一个很小的区域里，把“真房源 + 快响应 + 固定带看 + 供给侧付费”这四件事同时做好。

先赚第一笔钱，再把流程产品化。
