# 村圳表单搭建指南（飞书 / 微信承接）

更新日期：2026-03-09

## 1. 推荐最小方案

如果你想花时间最少，推荐组合：

- `飞书多维表 + 飞书表单`：做数据承接和日常管理
- `微信 / 企业微信`：做私信沟通和表单分发

不要一开始找人定制 CRM。

## 2. 一共只建 3 个表单

第一阶段够用的就是这 3 个：

1. 租客求租卡
2. 供给方房源提报表
3. 带看反馈表

## 3. 租客求租卡

### 表单标题

`村圳｜深圳西乡 / 福永租房求租卡`

### 表单简介

“填写后我会按你的预算、上班地和入住时间，优先给你匹配 `3` 套最近还能看的真房源。”

### 字段设计

- `姓名`：单行文本，必填
- `微信 / 手机`：单行文本，必填
- `预算下限`：数字，必填
- `预算上限`：数字，必填
- `上班地`：单选或下拉，必填
- `可接受通勤`：单选，选项：`30 分钟内 / 45 分钟内 / 60 分钟内`
- `房型偏好`：单选，选项：`单间 / 合租 / 一房`
- `是否接受合租`：单选，选项：`接受 / 不接受`
- `最晚入住时间`：日期，必填
- `是否晚归`：单选，选项：`经常 / 偶尔 / 很少`
- `是否有电动车`：单选，选项：`有 / 没有`
- `额外要求`：多行文本，选填
- `从哪里看到我`：单选，选项：`短视频 / 朋友介绍 / 社群 / 微信朋友圈 / 其他`

### 提交成功文案

“已收到。我会优先按你填的条件筛选房源，工作时间内尽量在 `10` 分钟内回复你。你也可以直接微信发我：`预算 + 上班地 + 房型`，我会更快处理。”

### 录入后的默认状态

- 线索状态：`新线索`
- 跟进优先级：
  - `A`：`7` 天内入住
  - `B`：预算和上班地明确
  - `C`：其余

## 4. 供给方房源提报表

### 表单标题

`村圳｜合作房源提报表`

### 表单简介

“仅接收真实在租、费用说明清楚、可提供视频的房源。资料不完整的房源不会发布。”

### 字段设计

- `联系人姓名`：单行文本，必填
- `联系人身份`：单选，选项：`房东 / 二房东 / 中介 / 拍房员`
- `微信 / 手机`：单行文本，必填
- `村名 / 片区`：单行文本，必填
- `房型`：单选，选项：`单间 / 合租 / 一房`
- `租金`：数字，必填
- `押付方式`：单行文本，必填
- `水电规则`：单行文本，必填
- `是否电梯`：单选，选项：`有 / 无`
- `楼层`：单行文本，必填
- `采光情况`：单选，选项：`好 / 一般 / 较差`
- `家具家电`：多行文本，选填
- `可入住时间`：日期，必填
- `视频链接`：链接，必填
- `最近核验时间`：日期时间，必填
- `是否接受成交后结算`：单选，选项：`接受 / 需沟通`
- `补充说明`：多行文本，选填

### 提交成功文案

“已收到。资料完整且核验通过后才会进入可发布池。如果字段不完整，我会联系你补充。”

### 自动规则

- 缺少 `租金 / 水电 / 核验时间 / 视频链接` 直接标记为 `待补充`
- 资料完整先进入 `待核验`
- 核验通过后才变为 `可发布`

## 5. 带看反馈表

### 表单标题

`村圳｜带看反馈表`

### 表单简介

“每次带看结束后当天回填，避免信息丢失。”

### 字段设计

- `带看日期`：日期，必填
- `预约编号`：单行文本，必填
- `租客姓名`：单行文本，必填
- `房源编号`：单行文本，必填
- `是否到场`：单选，选项：`到场 / 爽约`
- `如果爽约，原因`：多行文本，选填
- `租客满意度`：单选，选项：`高 / 中 / 低`
- `卡点原因`：多选，选项：`价格 / 通勤 / 环境 / 房态 / 临时失效 / 其他`
- `下一步动作`：单选，选项：`继续推进 / 补推 3 套 / 暂停跟进`
- `预计成交概率`：单选，选项：`高 / 中 / 低`
- `补充说明`：多行文本，选填

### 提交成功文案

“反馈已记录，今晚复盘时请优先处理高成交概率用户。”

## 6. 微信承接动作

表单不负责成交，微信负责成交。

### 推荐动作顺序

- 内容里放表单链接
- 用户填表后立刻微信跟进
- 已有明确预算和上班地的用户，先微信问 1 轮，再决定是否还需要补填表

### 微信首条自动口径

“收到，我先按你预算和上班地给你筛 `3` 套最近能看的。为了后面不重复问，你也可以顺手把求租卡补一下。”

## 7. 飞书多维表字段映射

### 求租卡表单 -> `lead-pipeline.csv`

- `姓名` -> `name`
- `微信 / 手机` -> `contact`
- `预算下限` -> `budget_min`
- `预算上限` -> `budget_max`
- `上班地` -> `work_area`
- `房型偏好` -> `room_preference`
- `最晚入住时间` -> `move_in_deadline`
- `从哪里看到我` -> `source_channel`

### 房源提报表 -> `listing-intake.csv`

- `村名 / 片区` -> `village`
- `房型` -> `room_type`
- `租金` -> `monthly_rent`
- `押付方式` -> `deposit_rule`
- `水电规则` -> `utility_rule`
- `楼层` -> `floor_label`
- `采光情况` -> `sunlight`
- `是否电梯` -> `elevator`
- `家具家电` -> `furniture_appliances`
- `可入住时间` -> `move_in_date`
- `最近核验时间` -> `last_verified_at`
- `视频链接` -> `video_url`
- `联系人身份` -> `source_role`

## 8. 表单搭建顺序

建议你明天只做这几个动作：

1. 先建租客求租卡
2. 再建供给方房源提报表
3. 最后建带看反馈表
4. 先跑一周，再补自动化和提醒

## 9. 不要现在就做的表单

- 在线签约表
- 在线支付表
- 房东自由发布注册表
- 复杂实名认证流程
- 多步骤长问卷

第一阶段表单的目标只有一个：`把数据收上来，并让你尽快成交`。
