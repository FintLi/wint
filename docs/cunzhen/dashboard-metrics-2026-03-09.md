# 村圳 60 天数据看板定义

更新日期：2026-03-09

## 1. 北极星指标

`每周成交单数`

因为你当前目标是快速赚钱，所以最重要的不是 DAU，也不是粉丝数，而是每周实际成交。

## 2. 四层漏斗

### 内容层

- 发布内容数
- 私信数
- 求租卡提交数

### 线索层

- 有效线索数
- 已联系数
- 已匹配数

### 带看层

- 已预约数
- 已确认数
- 已完成数
- 爽约数

### 成交层

- 已成交数
- 平均服务费
- 已结算金额
- 未结算金额

## 3. 每日必看指标

- 当天新增有效线索
- 当天首响时间
- 当天新增可发布房源
- 当天失效房源数
- 当天预约带看数

## 4. 每周必看指标

- 每周成交单数
- 每周带看完成率
- 每周爽约率
- 每周房源失效率
- 每周内容到求租卡转化率
- 每周求租卡到带看转化率
- 每周带看到成交转化率

## 5. 指标定义

### 有效线索

满足以下条件的求租卡：

- 预算明确
- 上班地明确
- 最晚入住时间明确
- 联系方式有效

### 首响时间

从用户首次私信 / 提交求租卡，到你第一次有效回复的时间。

### 房源失效率

`已失效房源数 / 可发布房源数`

### 内容到求租卡转化率

`求租卡提交数 / 私信数`

### 求租卡到带看转化率

`已完成带看数 / 有效线索数`

### 带看到成交转化率

`成交数 / 已完成带看数`

## 6. 目标值

### 第一个 30 天

- 首响时间：`<10` 分钟
- 房源失效率：`<20%`
- 带看完成率：`>70%`
- 带看到成交转化率：`>20%`

### 第一个 60 天

- 有效房源：`80`
- 有效线索：`150`
- 带看：`30`
- 成交：`8`
- 房源失效率：`<15%`

## 7. 每周复盘顺序

每周固定按这个顺序复盘：

1. 哪个内容带来的有效线索最多
2. 哪个村 / 预算带最容易约到看房
3. 哪个合作方失效率最高
4. 哪个环节掉单最多：联系、匹配、带看还是签约
5. 下周要砍掉什么，而不是只问要加什么

## 8. 数据记录建议

第一阶段直接用表格记录即可。

建议至少有 4 张表：

- 房源表
- 求租表
- 带看表
- 内容表

仓库里的 CSV 模板已经覆盖这 4 张表。
