# 工作选择系统设计说明

更新日期：2026-03-09

## 1. 设计定位

这不是一个“求职助手”，而是一个“工作选择系统”。

核心判断是：

- 市场上的工作机会并不稀缺，稀缺的是你能否持续发现它们
- 公开职位只是表层供给，真正高价值的工作机会往往分散在公开信号、组织动作、经营变化和人际网络中
- LLM 的主要作用不是代替你海投，而是帮助你把噪音很大的机会市场，压缩成一个可比较、可选择、可跟进的机会池

因此，这个系统的目标不是“提高投递量”，而是：

- 更早发现机会
- 更准识别高价值机会
- 更少消耗在低质量机会
- 更快做出选择动作

## 2. 产品目标

系统围绕一句话工作：

> 把分散在各处的工作信号，转成可被你持续选择的机会地图。

### 2.1 一级目标

- 建立稳定的机会发现机制，而不是临时刷招聘网站
- 把“钱多事少离家近”从口号变成评分模型
- 支持你每天只处理少量高价值机会，而不是浏览大量低质量岗位
- 让系统积累你的偏好，逐步学会你真正愿意选什么工作

### 2.2 二级目标

- 降低你对单一招聘平台的依赖
- 提升你发现隐性机会和半显性机会的能力
- 在正式投递前，先判断“值不值得接近”
- 为后续定向触达、提案式切入、内推等动作提供素材

## 3. 非目标

系统首期不追求以下内容：

- 不做无脑自动海投
- 不做全自动代聊 HR
- 不把所有抓到的岗位都塞给你
- 不把“匹配度”误当成“值得选择”
- 不承诺准确判断真实工作强度，只做基于公开信息的近似估计

## 4. 核心概念

## 4.1 Opportunity 而不是 Job

系统中的核心实体是 `Opportunity`，不是 `Job`。

`Opportunity` 指一切“值得你接近、评估、争取”的工作机会，包含三类：

- `显性机会`：已经公开发布的招聘岗位
- `半显性机会`：组织有明显扩张或补位信号，但岗位还未完全公开
- `隐性机会`：组织存在明确问题、缺口或增长任务，只是尚未被包装成正式职位

## 4.2 工作选择而不是求职

系统不是从“我缺一个岗位”出发，而是从“市场上有哪些值得我选的工作关系”出发。

因此，系统先回答这几个问题：

- 哪些组织现在更可能需要我这种人
- 哪些需求还没写成岗位，但已经客观存在
- 哪些岗位虽然公开，但实际不值得投入时间
- 哪些机会适合公开投递，哪些更适合绕开招聘入口定向触达

## 4.3 钱多事少离家近的系统化表达

“钱多事少离家近”在系统里要被拆成具体维度：

- `钱多`：现金薪资、总包上限、股权质量、薪酬透明度、兑现概率
- `事少`：职责堆叠程度、风险词密度、组织是否高压、值班/救火概率、角色边界是否清晰
- `离家近`：远程优先级、混合办公条件、门到门通勤时间、工作时区友好度
- `好拿下`：你的独特优势是否明显、竞争是否拥挤、是否存在低摩擦切入路径
- `老板正常`：团队表达清晰度、管理者公开风格、组织稳定性、招聘描述可信度

## 5. 系统总览

系统由四层组成：

- `采集层`：持续发现公开信号与半公开信号
- `理解层`：由 LLM 将信号解释为机会与风险
- `决策层`：按你的偏好进行估值、排序和推荐
- `执行层`：通过飞书和 OpenClaw 支持观察、触达、投递、跟进等动作

## 6. 技术架构

## 6.1 OpenClaw 的职责

`OpenClaw` 是总调度器和执行代理，负责：

- 定时任务调度
- 浏览器自动化与站点采集
- 消息流转与提醒
- 调用外部脚本与 API
- 审批流控制
- 多 Agent 协作
- 记忆与会话状态保留

## 6.2 飞书的职责

`飞书` 是数据与决策中台，负责：

- `Bitable` 存储机会池、公司库、信号库、跟进记录
- 飞书机器人推送日报、提醒、待审批项
- 人工在飞书里做 `观察 / 接近 / 忽略 / 触达` 决策
- 为后续复盘提供结构化数据入口

## 6.3 LLM 的职责

大模型主要做理解、估值和表达增强：

- 从网页、帖子、JD、社媒动态中抽取结构化信息
- 根据上下文推断机会层级与角色缺口
- 生成机会摘要、风险提示和切入建议
- 依据你的偏好计算排序分和解释分
- 生成定向触达文案、简历定制建议和面试准备材料

## 6.4 外部能力

系统可接入以下补充能力：

- `Playwright` 或浏览器抓取组件：处理动态页面和表单
- 地图与路线 API：估算真实通勤时间
- 邮件与日历：跟进触达和面试安排
- 搜索与新闻源：捕获融资、裁员、上线、扩张等组织信号

## 7. 多 Agent 设计

系统建议至少拆成四个 Agent。

## 7.1 雷达 Agent

职责：扫描外部信号，持续发现机会候选。

信号来源包括：

- 招聘平台与公司官网
- 创始人、招聘经理、团队负责人的社媒发言
- GitHub、产品更新日志、技术博客
- 投融资新闻、行业报道、社区讨论
- 内推群、招聘群、垂直社区和论坛

输出：

- 原始信号条目
- 相关公司与链接
- 初始机会猜测

## 7.2 解码 Agent

职责：把“信号”翻译成“机会”。

例如：

- 某公司连续发布 AI 自动化内容，但没有挂岗位，可能意味着缺流程自动化负责人
- 某产品刚完成海外上线并频繁修复问题，可能意味着缺平台、SRE、支持工程师
- 某团队创始人持续表达增长压力，可能意味着增长、数据、投放、内容自动化岗位正在形成

输出：

- `Opportunity` 草案
- 机会层级：显性 / 半显性 / 隐性
- 机会理由与证据链

## 7.3 估值 Agent

职责：判断这是不是值得你花时间的机会。

输出：

- 总体推荐分
- 钱多分
- 事少分
- 离家近分
- 好拿下分
- 老板正常分
- 一段可读解释：为什么值得看 / 为什么不值得看

## 7.4 触达 Agent

职责：给出进入路径，而不是机械生成投递文案。

可选路径：

- 公开投递
- 找朋友内推
- 定向发短邮件或私信
- 以“一页问题解决方案”切入
- 继续观察，等更强信号出现再动作

输出：

- 推荐动作
- 推荐理由
- 触达草稿
- 所需材料清单

## 9. 评分与估值模型

## 9.1 基础评分

首版建议采用可解释优先的线性模型：

`总分 = 0.35 * 钱多 + 0.25 * 事少 + 0.20 * 离家近 + 0.10 * 好拿下 + 0.10 * 老板正常`

说明：

- `钱多` 默认优先级最高，因为这是结果导向指标
- `事少` 放第二，但其本质只能做近似估计
- `离家近` 既包含物理通勤，也包含远程和时区友好度
- `好拿下` 代表机会成本，不只是简历匹配度
- `老板正常` 用来过滤大量低质量高薪陷阱

## 9.2 事少的近似规则

“事少”不能直接观测，只能代理估计。可以从以下信号推断：

- JD 里是否同时要求研发、产品、运营、支持、交付多种职责
- 是否出现 `抗压`、`快速响应`、`owner 意识`、`7x24`、`on-call` 等高压词
- 团队规模与预期产出的比例是否异常
- 公开渠道是否频繁表现出忙乱、救火、长期急招
- 岗位描述是否边界模糊

## 9.3 离家近的实际定义

“离家近”不是地图直线距离，而是综合决策成本：

- 纯远程为最高分
- 混合办公看到岗频次和时区要求
- 线下岗位按门到门通勤时间计分
- 若需要跨城迁移、长期出差、倒时差，则显著扣分

## 10. 日常工作流

## 10.1 发现流程

- `OpenClaw` 按设定频率抓取来源
- 雷达 Agent 写入 `Signals`
- 解码 Agent 将强信号转成 `Opportunities`
- 估值 Agent 打分并生成解释
- 飞书日报推送当日 Top 机会

## 10.2 决策流程

你在飞书里只需要做少量动作：

- `观察`
- `接近`
- `忽略`
- `转人工深挖`

系统不要求你当场投递，而是先判断值不值得投入更多时间。

## 10.3 触达流程

当某个机会被标记为 `接近` 时：

- 触达 Agent 生成推荐进入路径
- LLM 生成与该路径对应的文本素材
- 若需要，可再生成定制简历或一页提案
- 最终动作建议保留人工审批

## 11. 飞书视图设计

飞书里建议至少有四个视图：

- `今日机会`：今天新增且分数高的机会
- `观察池`：还需要更多证据的机会
- `高分可接近`：适合近期动作的机会
- `已放弃复盘`：沉淀你为什么不选

日报消息建议只推送少量内容：

- Top 5 新机会
- 2-3 条需要你决定的高价值项
- 1 条模型不确定但值得你看一眼的异常项

## 12. MVP 范围

## P0：先证明这套逻辑成立

只做这三件事：

- 抓取公开岗位和公司官网职位
- 用 LLM 做结构化和基础评分
- 用飞书每天推送 Top 机会

## P1：加入半显性机会发现

- 扫描创始人、招聘经理和团队动态
- 识别扩张、补位、增长、工程压力等组织信号
- 让系统开始产生“可能存在但尚未公开”的机会

## P2：加入触达策略

- 为高分机会生成进入路径
- 生成简短私信、邮件或一页提案
- 记录哪种路径成功率更高

## P3：加入偏好学习

- 根据你每次的 `观察 / 接近 / 忽略` 决策迭代模型
- 逐步修正权重和推荐风格
- 让系统更贴近你对“好工作”的真实定义

## 13. 风险与约束

需要明确的现实约束：

- 招聘平台可能对自动化抓取和自动投递敏感
- LLM 对“工作强度”“老板正常”这类维度只能做近似推断
- 半显性与隐性机会判断容易出现误报，需要通过证据链约束
- 如果缺少人工审批，系统可能过度自信地推荐低质量动作

因此首期设计强调：

- 高可解释
- 保留人工审批
- 推荐少而精
- 先发现，再动作

## 14. 成功指标

这个系统是否有效，不看“抓到多少岗位”，而看：

- 每周新增高价值机会数
- 你实际愿意接近的机会占比
- 通过非公开路径进入沟通的比例
- 你花在低质量岗位上的时间是否下降
- 最终拿到的机会，是否更符合“钱多事少离家近”

## 15. 结论

这套方案的关键不是把 LLM 变成简历机器，而是把它变成工作市场解释器。

你的动作重心也不是“多申请”，而是：

- 建立自己的机会雷达
- 用模型压缩信息噪音
- 用结构化数据沉淀判断
- 用飞书做每天的小决策
- 把真正高价值的工作，从信息流里主动挑出来

一句话总结：

> 这不是 Job Search OS，而是 Work Selection OS。
